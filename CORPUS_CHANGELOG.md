# CORPUS_CHANGELOG.md ‚Äî frozen-corpus revision protocol

The test corpus is generated by `generate_fixtures.m` (seed 20260717) and frozen: every
fixture `.mat` is SHA-256'd into `fixtures/manifest.json`. Auditability is the deliverable
‚Äî so **any** change to the corpus is a logged EVENT here, with the new `corpus_version`
(also stamped into the manifest), what changed, why, the count delta, and the fact that
all hashes + manifest were regenerated. A reader comparing "N fixtures" across artifacts
must be able to reconcile every number from this file.

Regeneration always re-hashes ALL files (fixture structs gained fields across versions,
so hashes legitimately change between versions ‚Äî not just the added files).
